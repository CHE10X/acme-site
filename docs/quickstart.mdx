## I need to diagnose a problem

**Start with ACME Triage.** Free, open source, zero commitment. It scans your agent environment and tells you exactly what's wrong.

```bash
curl -fsSL https://acmeagentsupply.com/install/triage | bash
```

Triage is your diagnostic starting point — it identifies the problem, then the ACME paid stack fixes it.

<Accordion title="What does Triage examine?">
- **Stall risk** — patterns that indicate potential hangs
- **Silence gaps** — unexpected inactivity windows
- **Compaction pressure** — context window saturation
- **Configuration hygiene** — common reliability pitfalls

Nothing is modified. Read-only.
</Accordion>

**Interpret your score:**

| Score | Status | What It Means |
|-------|--------|---------------|
| 80–100 | 🟢 GREEN | Healthy. Low immediate risk. |
| 60–79 | 🟡 YELLOW | Watch closely. Some risk factors present. |
| 40–59 | 🟠 ORANGE | Attention needed. Act before it gets worse. |
| 0–39 | 🔴 RED | Critical. High risk of silent failure. |

## I want runtime protection

**Sentinel** monitors your agent continuously — catching stalls, silent failures, and compaction events before they become incidents.

```bash
curl -fsSL https://acmeagentsupply.com/install/sentinel | bash
```

Sentinel runs as a background process. It alerts and provides context — it doesn't take autonomous action.

→ [Sentinel documentation](/products/sentinel/overview)
