## What You're Setting Up

Three persistent services that run on your machine and connect your local agent workspace to the ACME control plane:

| Service | Port | What it does |
|---------|------|-------------|
| **QM v2** | 8090 | Ingests events, tracks mission health, manages credits |
| **Dispatch v3** | 8091 | Auto-dispatches tasks to agents on a 5-second scheduler loop |
| **INBOX Watcher** | — | Detects `[UNREAD]` items in agent INBOXes and triggers agent wakes |

Once connected, the loop is automatic:

```
You add a task to an agent's INBOX
  ↓
INBOX Watcher detects it (within 30s)
  ↓
QM v2 ingests the event, updates health state
  ↓
WakeState bridge transitions agent: IDLE → WAKING → ACTIVE
  ↓
Agent wakes and processes the task
```

## Prerequisites

- macOS (launchd required for persistent services)
- Python 3.10+ (verify: `which python3` — note the path for plist files)
- OpenClaw installed and gateway running
- ACME workspace cloned at `~/.openclaw/workspace/`
- `quartermaster_v2/` and `dispatch_v3/` modules present in workspace

<Note>
The plist examples below use `/Users/AGENT/` as the home directory. **Replace all instances with your actual home directory path.**

</Note>

Verify your workspace:
```bash
ls ~/.openclaw/workspace/quartermaster_v2/
ls ~/.openclaw/workspace/dispatch_v3/
```

## Step 1 — Apply Credit Engine Fix

The credit engine requires a threading fix before QM v2 will run reliably.

Edit `~/.openclaw/workspace/quartermaster_v2/state/credit_engine.py`:

```python
# Find this line:
self._lock = threading.Lock()

# Replace with:
self._lock = threading.RLock()
```

Also set credit allowances to unlimited (dynamic allocation replaces fixed caps):

```python
DEFAULT_ALLOWANCES = {
    "ops_head": float("inf"),
    "active_mission": float("inf"),
    "standard": float("inf"),
    "background": float("inf"),
    "cron": float("inf"),
}
```

## Step 2 — Start QM v2

```bash
pkill -9 -f "uvicorn.*quartermaster" 2>/dev/null || true

cd ~/.openclaw/workspace && \
  PYTHONPATH=. nohup python3 -m uvicorn \
    quartermaster_v2.api.server:create_app \

    --host 127.0.0.1 \
```

## Step 7 — Enable Agent Task Claiming on Heartbeat

Agents need to poll Dispatch v3 for tasks on every heartbeat. Add the following to the top of each agent's `HEARTBEAT.md` (before the INBOX check):

```markdown
## Dispatch Claim (every heartbeat — first)
1. Run: `curl -s -X POST http://127.0.0.1:8091/dispatch/v3/claim/{your_agent_id}`
2. If `claimed: true` → execute the task based on `task_type` and `payload`
3. When done: `curl -s -X POST http://127.0.0.1:8091/dispatch/v3/finish/{task_id}?success=true`
4. If `claimed: false` → no pending tasks, continue to INBOX check
```

Or use the agent handler CLI directly:

```bash
# Claim next task for an agent
python3 -m dispatch_v3.agent_handler claim --agent <agent_id> --json
# Returns: {"claimed": true, "task_id": "...", "task_type": "...", "payload": {...}}

# Report task complete
python3 -m dispatch_v3.agent_handler finish --agent <agent_id> --task-id <task_id> --json
# Returns: {"finished": true}

# List pending tasks
python3 -m dispatch_v3.agent_handler tasks --agent <agent_id> --json
```

## Step 8 — Test the Full End-to-End Loop

Submit → auto-dispatch → agent wakes → agent claims → agent finishes:

```bash
# 1. Submit a task
curl -s -X POST http://127.0.0.1:8091/dispatch/v3/submit \
  -H "Content-Type: application/json" \
  -d '{"agent_id":"soren","task_type":"inbox_check","payload":{"reason":"setup_test"}}'

# 2. Dispatch v3 scheduler auto-dispatches within 5s
#    (or trigger manually:)
curl -s -X POST "http://127.0.0.1:8091/dispatch/v3/dispatch?agent_id=soren"

# 3. Confirm agent was woken (targeted, not broadcast)
tail -3 ~/.openclaw/wakestate/audit_journal.jsonl | grep soren
# Expected: IDLE → WAKING → ACTIVE

# 4. Claim the task
python3 -m dispatch_v3.agent_handler claim --agent soren --json

# 5. Finish the task
python3 -m dispatch_v3.agent_handler finish --agent soren --task-id <task_id> --json

# 6. Verify in QM
curl -s "http://127.0.0.1:8090/qm/events/recent?limit=5"
```

## Troubleshooting

**Service won't start:**
```bash
# Check logs
tail -30 ~/.openclaw/watchdog/qm_v2.log
tail -30 ~/.openclaw/watchdog/dispatch_v3.log
tail -30 ~/.openclaw/watchdog/inbox_watcher.log
```

**Port already in use:**
```bash
lsof -i :8090   # find what's on 8090
lsof -i :8091   # find what's on 8091
```

**INBOX watcher not detecting items:**
```bash
# Delete state file to reset detection
rm ~/.openclaw/qm/inbox_watcher_state.json

# Run once in foreground to see output
cd ~/.openclaw/workspace && PYTHONPATH=. python3 -m quartermaster_v2.inbox_watcher --once
```

**WakeState not transitioning:**
```bash
# Check WakeState audit journal directly
cat ~/.openclaw/wakestate/audit_journal.jsonl | tail -20

# Manual wake test
curl -s -X POST "http://127.0.0.1:8090/qm/wake/soren?task_id=test&reason=debug"
```
