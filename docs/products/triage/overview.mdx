# triage

Before you touch anything, run `triage`.

It captures a read-only snapshot of your stack — gateway health, session state, disk pressure, compaction history, and more — and saves it with a timestamp. That snapshot is your evidence. You want it before you start changing things.

<Info>
`triage` is free, MIT licensed, and requires no account. It never modifies your system.
</Info>

## Install

```bash
curl -fsSL https://acmeagentsupply.com/install/triage | bash
```

## Run

```bash
triage
```

Done in seconds. Output lands at `~/.openclaw/triage-bundles/YYYY-MM-DD-HH-MM-SS/`.

## What It Captures

Seven checks run every time:

| Collector | What It Grabs |
|-----------|--------------|
| `gateway` | Liveness, port status, probe response |
| `sessions` | Active session count and topology |
| `disk` | Log volume, growth rate, pressure |
| `integrity` | Binary and config file checks |
| `doctor` | Environment and dependency health |
| `compaction` | Compaction event history and current budget |
| `fleet` | Version, uptime, environment identity |

## Reading Your Bundle

Open these first:

```
gateway_status.txt      — is the gateway alive and actually responding?
compaction_status.txt   — any active memory pressure?
fleet_identity.txt      — version, uptime, confirmed
disk_pressure.txt       — log volume and growth rate
```

A clean bundle means you know what "before" looks like. Now you can make changes and compare.

## What triage Does NOT Do

- No recovery or restart — ever
- No writes to your config
- No background processes or startup items installed
- No network calls
