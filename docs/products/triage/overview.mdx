# triage

Before you touch anything, run `triage`.

It captures a snapshot of your stack right now — gateway health, session state, disk pressure, memory history — and saves it with a timestamp. That snapshot is your evidence. You want it before you start changing things.

<Info>
Free, open source, no account needed. Never modifies your system.
</Info>

---

## Install

**Step 1 — Open your terminal**

On Mac: Press `Cmd + Space`, type `Terminal`, press Enter.  
On Windows: Press `Win + R`, type `cmd`, press Enter.

**Step 2 — Paste this and press Enter**

```bash
curl -fsSL https://acmeagentsupply.com/install/triage | bash
```

**What you'll see:**

```
Installing triage...
✓ Cloned repository
✓ Installed to /usr/local/bin/triage
triage 0.1.3 ready
```

If you see `Permission denied`, run it with `sudo` in front:
```bash
sudo curl -fsSL https://acmeagentsupply.com/install/triage | bash
```

<Tip>
Not sure what any of this means? Paste the error message into Claude or ChatGPT and ask — that's exactly what they're there for.
</Tip>

---

## Run It

**Step 3 — Run triage**

```bash
triage
```

That's it. It runs, collects everything, and saves a bundle. Takes about 10 seconds.

**What you'll see:**

```
triage 0.1.3
───────────────────────────────
✓ gateway         alive, responding
✓ sessions        2 active
✓ disk            normal
✓ integrity       all checks passed
✓ doctor          environment ok
✓ compaction      no active pressure
✓ fleet           openclaw 2026.3.11, uptime 4h

Bundle saved → ~/.openclaw/triage-bundles/2026-03-15-22-00-00/
───────────────────────────────
```

## Reading Your Results

Your bundle is a folder of plain text files. Open it in Finder (Mac) or File Explorer (Windows), or read them in the terminal.

**The files that matter most:**

```
gateway_status.txt     — is the gateway actually responding?
compaction_status.txt  — any memory pressure building up?
fleet_identity.txt     — version, uptime, confirmed
disk_pressure.txt      — how much log space are you using?
```

**Green across the board?** Your stack is healthy right now. Keep the bundle — it's your "before" picture if something breaks later.

**Something showing as a problem?** That's normal for a first run. It's not an emergency — it's information. Check the [RadCheck overview](/docs/products/radcheck/overview) for what to do next, or paste the output into Claude and ask what it means.

---

## What triage Does NOT Do

- It never restarts anything
- It never changes your config
- It never sends your data anywhere
- It never installs background services

If something breaks after running triage, triage didn't cause it.

---

## If Something Goes Wrong

**"command not found" after installing:**
Close your terminal, open a new one, try again. The installer updated your PATH but the current window doesn't know yet.

**"Permission denied" during install:**
```bash
sudo curl -fsSL https://acmeagentsupply.com/install/triage | bash
```

**"git: command not found":**
You need Git installed first. On Mac: `xcode-select --install`. Then try the triage install again.

**Still stuck?** [Contact support](/docs/support/contact) or paste your error into Claude/ChatGPT.
