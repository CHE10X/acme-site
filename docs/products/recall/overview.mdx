# Recall

Sometimes you need to reach in and stop something. Recall is how you do that.

It's a CLI for direct intervention — pausing agents, isolating broken ones, and controlling your stack when things aren't going the way monitoring expected.

```bash
recall status          # what's the current state of everything
recall stall heike     # pause an agent right now
recall wake heike      # resume it
recall lockdown        # stop everything, immediately
```

## The Three Things You'll Actually Use

Most situations call for one of three commands:

| Command | What It Does | When to Use It |
|---------|-------------|----------------|
| `recall stall <agent>` | Pauses the agent — stops it from acting, keeps it listening | First instinct when something looks wrong |
| `recall focus <agent>` | One agent acts, everyone else pauses | Two agents stepping on each other |
| `recall stun <agent>` | Full intervention — session archived, sub-agents killed, holds for your review | Something is genuinely broken |

**`stall` is the right first move almost every time.** Most problems are coordination failures, not catastrophic ones. Stop the agent, look at what happened, then decide whether to `wake` (minor) or `stun` (broken).

## Full Command Reference

**System-wide:**
```bash
recall lockdown        # no new model calls, no tool calls, no spawning
recall unlock
recall status          # fleet snapshot
recall log [agent]     # history of what happened
```

**Per-agent state:**
```bash
recall freeze <agent>  # block sub-agent spawning only
recall unfreeze <agent>
recall stall <agent>   # pause message processing
recall stall --all     # stop everything
recall wake <agent>    # resume
recall sleep <agent>   # disconnect from channels
recall stun <agent>    # full intervention (see below)
recall quarantine <agent>
```

**Recovery:**
```bash
recall recover <agent> # guided return-to-service after stun
recall reset           # safe gateway restart (requires backup verification)
recall focus <agent>   # single-leader mode
recall unfocus         # back to normal
```

## What Stun Actually Does

`recall stun` isn't a restart. It's a 7-step intervention:

1. Disconnect channel connections
2. Drain in-progress tool calls (max 10 seconds)
3. Kill all sub-agents
4. Archive the active session
5. Compact memory
6. Emit an event (visible in your monitoring)
7. Hold — agent stays offline until you run `recall recover`

**Why not just restart the gateway?** A restart doesn't fix a runaway agent. The bad state lives in the session, the sub-agents, and the memory files. Stun targets all three. Restart misses all three.

## Everything Reverses

| Command | Reversal |
|---------|---------|
| `lockdown` | `unlock` |
| `freeze` | `unfreeze` |
| `stall` | `wake` — queued messages delivered |
| `sleep` | `wake` |
| `stun` | `wake` after `recover` + your confirmation |

Session context after stun isn't restored — by design. If the session was the problem, restoring it restores the problem.

## Standard Incident Flow

```bash
openclaw watch         # notice something's off
recall stall heike     # pause immediately
triage                 # capture a snapshot before touching anything
agent911 diagnose      # what actually happened

# If it's minor:
recall wake heike

# If something's broken:
recall stun heike
recall recover heike
recall wake heike
```
