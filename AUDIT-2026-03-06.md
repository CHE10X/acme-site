## SITE-WIDE — Visual Character Regression

These issues affect every page and explain why the site feels "generic template" rather than ACME.

### VR-01 · Body text is too small — minimum 18px not enforced
**Spec:** VL2 §Typography — body text minimum 18px, card body 17px, nothing below 13px  
**What's wrong:** Most descriptive text across all pages appears to be 14-15px. Platform page and Support page body text is visibly small. Makes the site feel dense and low-confidence.  
**Action needed:** Audit all `p`, `li`, `.node-desc` text across all pages. Enforce 18px minimum for body, 17px minimum for card descriptions.

### VR-02 · Page backgrounds are near-black (#0D1117 range), not spec
**Spec:** VL2 — page bg `#1E2226`, section panels `#242A30`  
**What's wrong:** The current site appears to use near-black (likely Tailwind `bg-gray-950` or similar). This makes everything feel flat, dark, and generic — loses the depth contrast between bg and panels that gives the site its field instrument feel.  
**Action needed:** Update `app/globals.css` — set body/page background to `#1E2226`. Section panels to `#242A30`. Inset panels to `#161A1E`.

### VR-03 · Section panels missing — no visual layer differentiation
**Spec:** VL2 — sections should use panel background `#242A30` with `border: 1px solid #3A4048` on all sides  
**What's wrong:** Most page sections have no background fill or border, so they blend into the page background. The page becomes one undifferentiated dark surface. This is the single biggest reason the site lost its visual character.  
**Action needed:** Every major section (hero, terrain, reliability stack, OCTriage, etc.) should sit inside a panel with `background: #242A30`, `border: 1px solid #3A4048`, `border-radius: 4-6px`. Not every paragraph — sections.

### VR-04 · Amber used inconsistently and decoratively
**Spec:** VL2 — amber `#D98A2B` is ONLY for: primary CTAs, diagram flow highlights, warning/alert states. Never decorative.  
**What's wrong:** On the Platform page, product names like `Elixir`, `DriftGuard`, `FindMyAgent`, etc. are rendered in amber as inline code spans. This dilutes amber's signal value — it looks like syntax highlighting, not intentional brand color.  
**Action needed:** Product names in body text should be white or light (#E6E6E6) or styled as a non-amber chip. Reserve amber for actual CTAs and actual status states only.

### VR-05 · Card borders missing or inconsistent
**Spec:** VL2 — card borders are deterministic all-sides: `1px solid #3A4048`  
**What's wrong:** Many cards across the site have no visible border, or have borders only on one side (top-only rule visible on some elements). Cards without borders on the current dark background become invisible blobs.  
**Action needed:** All card elements: `border: 1px solid #3A4048` all sides, `border-radius: 6px`, `background: #2C3238`.

### VR-06 · No visual hierarchy between eyebrow / title / body
**Spec:** VL2 — eyebrow labels: 10-11px uppercase, letter-spacing 0.14em, color `#4A5E70`; titles: 48/56px hero, 22-26px section; body: 18px  
**What's wrong:** Across most pages, there are no eyebrow labels. Section titles and body text are similar sizes, which creates a flat reading experience with no visual rhythm. The site reads like a markdown doc, not a product site.  
**Action needed:** Add eyebrow labels to all major sections (e.g., `ACME AGENT SUPPLY CO.`, `RELIABILITY STACK`, `OPERATOR CONSOLE`). Enforce the font-size hierarchy throughout.

## DECISIONS NEEDED (Flag for Chip)

### D-01 · Product logos on product pages — keep, resize, or remove?
Chip flagged logos as displaced. Need decision before Codex touches product pages:
- **Option A:** Remove logos entirely — let product name + description carry the card
- **Option B:** Resize and strictly constrain to 32×32px or 48×48px in top-left of card
- **Option C:** Use a single consistent icon system (simple geometric mark per product)

Recommendation: Option A (remove) unless there's a consistent icon set ready. Misaligned logos hurt more than no logos.

### D-02 · Bot Shop — full rebuild scope
The Bot Shop is broken enough that it needs a full rebuild spec before Codex touches it. Should it be:
- **Option A:** Product grid (cards), each with image, name, price, Add to Cart
- **Option B:** Categorized list (Emergency, Diagnostics, Operator) with pricing table
- **Option C:** Keep current category structure but fix the rendering

Recommendation: Option B matches the architecture Chip previously described. Need confirmation before directing Codex.

### D-03 · Homepage terrain — iframe vs inline SVG
Codex needs a clear directive on how to embed operator-terrain-v2.html:
- **Option A:** Full-bleed `<iframe>` with `scrolling="no"`, height=620px — simplest but risks scroll trapping
- **Option B:** Extract SVG+CSS+JS from the HTML and inline it directly into the React component — cleanest but requires careful extraction
- **Option C:** Serve as a static page and embed via iframe with `pointer-events` handled in CSS

Recommendation: Option B. I can produce an extracted SVG-only version for Codex to drop in.
